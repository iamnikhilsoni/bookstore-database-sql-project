# 📊 Bookstore Database SQL Analysis Project

## 📌 Brief One Line Summary
A SQL-based project to analyze bookstore data and extract insights on sales, customers, and inventory.

## 🔍 Overview
This project focuses on analyzing a bookstore database using SQL to understand business performance. It explores patterns in book sales, customer purchasing behavior, and inventory management. The queries help in deriving meaningful insights that can support data-driven decision-making.

## ❗ Problem Statement
Businesses often struggle to analyze raw transactional data and extract actionable insights. This project aims to simplify that process by using SQL queries to uncover trends in sales, customer activity, and stock management.

## 📂 Dataset
The project uses a structured relational database consisting of:

- 📖 Books: Book details such as title, author, genre, price, and stock  
- 👤 Customers: Customer information including name, city, and country  
- 🛒 Orders: Transactional data including quantity, total amount, and order date  

## 🛠️ Tools and Technologies
- SQL (PostgreSQL / MySQL)
- Relational Database Concepts
- Git & GitHub

## ⚙️ Methods
- Data Filtering (WHERE clause)
- Data Joining (INNER JOIN, LEFT JOIN)
- Aggregation (SUM, AVG, COUNT)
- Grouping & Filtering (GROUP BY, HAVING)
- Sorting & Limiting Results (ORDER BY, LIMIT)
- Data Deduplication (DISTINCT)

## 📈 Key Insights
- Identification of top-selling books  
- Most valuable customers based on spending  
- Sales distribution across genres and authors  
- Revenue generated from orders  
- Cities with high-value customers  
- Remaining stock after fulfilling orders  
- Frequently ordered books  

## 🖥️ Project Output
The SQL analysis provides:
- Sales insights (total revenue, top books)
- Customer insights (high spenders, order frequency)
- Inventory insights (stock availability, low stock books)
- Genre and author performance analysis
